# TPS Calendar Base 0.2.3

## Intent

Keep Calendar day numbers readable in narrow desktop, split-pane, and mobile layouts.

## User-visible changes

- Removed the aggregate checklist/count control from Calendar day headers.
- Returned the control's reserved header width to the native weekday/date label so the numeric day remains visible at constrained widths.
- Preserved the separate auxiliary-date and archived-external warning markers and their read-only menus.

## Compatibility and migration

- Minimum Obsidian version: `1.10.0`.
- No settings, note, or data migration is required.
- The repository is public and can be installed or updated through BRAT without a GitHub token.

## Validation

- Focused day-header regression: 5/5 passed.
- Complete exact-release declared test suite: 69/69 passed, including TypeScript and the suite build.
- Required separate production-mode build passed and atomically deployed `main.js`, `manifest.json`, and `styles.css` to the test-vault runtime.
- The deployed runtime matched the exact tested release artifact byte-for-byte; no plugin-owned `data.json` existed to replace.
- Test-vault reload and constrained Calendar UI verification passed in Obsidian 1.12.7: the 3-day QA Calendar showed complete numeric labels for July 16–18 and exposed no aggregate day-context controls.
- This release is for public BRAT distribution and has not been directly deployed to the production vault by this task.
