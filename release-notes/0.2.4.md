# TPS Calendar Base 0.2.4

This patch prevents Calendar settings from replacing synchronized choices with stale whole-file snapshots.

## Changes

- Reloads current plugin data and merges only locally changed known keys, preserving unrelated synchronized and unknown/future fields.
- Retains rapid change/revert intent, makes callers wait for the full durable drain, and lets a queued newer value supersede an earlier failure.
- Preserves intentionally empty style rules and all supported color targets; text edits persist without unload-unsafe debounce timers.

## Compatibility and validation

- Minimum Obsidian: 1.10.0; desktop and mobile supported; no manual migration.
- Focused settings/create/mobile tests: 31/31 passed; TypeScript and diff checks passed.
- Full declared suite: 75/75 passed. A separate production-mode build deployed byte-identical artifacts to the isolated test vault.
- Obsidian 1.12.7 was reloaded with **Reload app without saving**; Calendar registered normally and no runtime settings file was created or replaced.
- GitHub Actions standalone builds remain unavailable because this contained layout imports the adjacent shared deployment helper.
