# TPS Calendar Base 0.2.4

This patch prevents Calendar settings from replacing synchronized choices with stale whole-file snapshots.

## Changes

- Reloads current plugin data and merges only locally changed known keys, preserving unrelated synchronized and unknown/future fields.
- Retains rapid change/revert intent, makes callers wait for the full durable drain, and lets a queued newer value supersede an earlier failure.
- Preserves intentionally empty style rules and all supported color targets; text edits persist without unload-unsafe debounce timers.
