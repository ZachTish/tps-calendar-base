# TPS Calendar Base 0.2.2

## Intent

Make TPS Calendar Base installable and updateable through BRAT while keeping its GitHub repository private.

## User-visible changes

- Fresh BRAT installs receive the complete Calendar and shared settings UI styling through the standard `styles.css` asset.
- Plugin startup no longer reads or injects a separate `styles-ui.css` file.
- Existing Calendar behavior, settings, data, and minimum Obsidian compatibility are unchanged.

## Compatibility and migration

- Minimum Obsidian version: `1.10.0`.
- No settings or data migration is required.
- BRAT private-repository access requires a fine-grained GitHub token with `Contents: Read-only` for `ZachTish/tps-calendar-base`.
- `styles-ui.css` remains in source and local deployments as the maintained input for the generated standard stylesheet; BRAT only needs `main.js`, `manifest.json`, and `styles.css`.

## Validation

- Focused BRAT release compatibility regression: 4/4 passed.
- Complete clean-release declared test suite: 69/69 passed, including TypeScript and the suite build.
- Required separate production-mode build passed; the final test-vault deployment reported `unchanged` after deploying the same artifact.
- Test-vault reload and UI verification: passed in Obsidian 1.12.7; the Calendar settings screen rendered its cards and collapsible sections from the standard `styles.css` artifact.
- This release is for test-vault/BRAT distribution and has not been promoted to production by this task.

## Known limitations

- BRAT support for private repositories is experimental and its read-only token must be configured separately on each device/vault.
- The repository's contained build still depends on the adjacent `Plugin Development/deploy-runtime.mjs`; GitHub Actions release automation remains intentionally disabled.
